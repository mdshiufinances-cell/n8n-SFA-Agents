## Tool 4 — Read Yardi Files: Architecture (v2.5)

**Status: ✅ Production-tested**

### Node Structure

```
[When Executed by Another Workflow]
        ↓
[Search files and folders]   (Always Output Data: ON)
        ↓
[Match File]                  (Code node v2.5, Always Output Data: ON)
        ↓
[IF: found === true]
   ↙ true                  ↘ false
[Download file]         [Not Found Message]  (Code node, final node)
   ↓
[Extract from File]   (Always Output Data: ON)
   ↓
[Code in JavaScript]   (formats success output)
```

### Trigger Input Schema (case-sensitive)

```
PropertyCode  (String)   — e.g. prop01
FileType      (String)   — actuals | budget | rent_roll | actuals_py | budget_py | rent_roll_py
```

⚠️ n8n expressions are case-sensitive. All downstream references MUST use `$json.PropertyCode` not `$json.propertyCode`.

### Search files and folders config
| Parameter | Value |
|---|---|
| Search Query | `{{ $json.PropertyCode }}` |
| Filter | Folder = yardi-input |
| Always Output Data | ON |

### Match File (Code node v2.5)

```javascript
const triggerData = $('When Executed by Another Workflow').first().json;
const fileType = triggerData.FileType;
const propertyCode = triggerData.PropertyCode;

const items = $input.all();

const patternMap = {
  actuals:      { base: 'actualbudget', yearSuffix: null },
  budget:       { base: 'monthbudget',  yearSuffix: null },
  rent_roll:    { base: 'rentroll',     yearSuffix: null },
  actuals_py:   { base: 'actualbudget', yearSuffix: '2025' },
  budget_py:    { base: 'monthbudget',  yearSuffix: '2025' },
  rent_roll_py: { base: 'rentroll',     yearSuffix: '2025' },
};

const config = patternMap[fileType];
if (!config) {
  return [{ json: { found: false, propertyCode, fileType,
    message: `Unknown FileType: ${fileType}` } }];
}

const { base: pattern, yearSuffix } = config;

const match = items.find(item => {
  const name = (item.json.name || '').toLowerCase().replace(/_/g, '').replace(/\s/g, '');
  if (!item.json.id) return false;
  if (!name.includes(pattern)) return false;
  if (yearSuffix) {
    return name.includes(yearSuffix);
  } else {
    const hasPYSuffix = name.includes('2025') || name.includes('2024') || name.includes('2023');
    return !hasPYSuffix;
  }
});

if (match) {
  return [{ json: {
    found: true, id: match.json.id, name: match.json.name, propertyCode, fileType
  }}];
}
return [{ json: { found: false, propertyCode, fileType } }];
```

### Not Found Message (Code node, FALSE branch)

```javascript
const j = $input.first().json;
return [{
  json: {
    output: JSON.stringify({
      success: false,
      message: `No ${j.fileType} file found for property ${j.propertyCode}. ` +
               `Confirm file is in /yardi-input/ with correct naming convention.`
    })
  }
}];
```

### Code in JavaScript (TRUE branch, final node)

```javascript
const items = $input.all();

if (!items || items.length === 0 || !items[0].json || Object.keys(items[0].json).length === 0) {
  const propertyCode = $('When Executed by Another Workflow').first().json.PropertyCode;
  const fileType = $('When Executed by Another Workflow').first().json.FileType;
  return [{ json: { output: JSON.stringify({
    success: false,
    message: `No ${fileType} file found for property ${propertyCode}.`
  }) } }];
}

const fileType = $('When Executed by Another Workflow').first().json.FileType || 'actuals';
const rows = items.map(item => item.json);
const isPY = fileType.endsWith('_py');

const keyGLCodes = ['41100','41110','41200','41990','42100','42990',
  '43100','43290','43510','44100','44990','45990',
  '61050','62000','63000','64000','65000','66000','69990',
  '75990','87990','89190','89290','89390','90000','90010'];

let outputRows;
if (fileType === 'actuals' || fileType === 'budget') {
  const filtered = rows.filter(row => {
    const gl = String(Object.values(row)[0] || '').trim();
    return keyGLCodes.includes(gl) || gl.endsWith('990') || gl.endsWith('000');
  });
  outputRows = filtered.length > 0 ? filtered : rows.slice(0, 100);
} else if (isPY) {
  // PY: return key GL subtotals only — annual totals for YOY comparison
  const filtered = rows.filter(row => {
    const gl = String(Object.values(row)[0] || '').trim();
    return keyGLCodes.includes(gl);
  });
  outputRows = filtered.length > 0 ? filtered : rows.slice(0, 80);
} else {
  outputRows = rows.slice(0, 200);
}

return [{ json: { output: JSON.stringify({
  success: true, fileType, rowCount: outputRows.length, data: outputRows
}) } }];
```

## Tool 5 — Generate Excel: Architecture (v2.0)

**Status: ✅ Production-tested — v4 workbook design**

### Node Structure

```
[When Executed by Another Workflow]
        ↓
[Code: Excel Workbook Generator v4]   (SheetJS, 5 tabs)
        ↓
[Google Drive — Upload file]          (saves to yardi-output, Always Output Data: ON)
        ↓
[Format Output]                       (final node — returns output JSON string)
```

### Trigger Input Schema

```
property      (String)   — e.g. prop01
period        (String)   — e.g. 2026-Q3
forecastData  (String)   — JSON string per forecastData schema v2
```

### Workbook v4 Tab Structure

| Tab | Purpose |
|---|---|
| Forecast | Main P&L — per-month scenario dropdowns L5/M5/N5; actuals Jul–Mar; forecast Apr–Jun |
| Overrides | Full-year mirror layout; yellow L/M/N input cells; GL-keyed lookup |
| Assumptions | 13 editable scenario % rows driving all forecast calculations |
| Key Findings | Executive dashboard — KPIs, variances, risks, opportunities, management checklist |
| _PY_Data | Hidden — FY2025 annual totals by GL (col O, row 12+); wired to Forecast col P |

### Override Formula Pattern (Forecast!L{r})

```
=IF(AND($A{r}<>"",
     COUNTIFS(Overrides!$A$12:$A$400,$A{r},Overrides!$L$12:$L$400,"<>")>0),
   SUMIFS(Overrides!$L$12:$L$400,Overrides!$A$12:$A$400,$A{r}),
   IFERROR(AVERAGE(I{r}:K{r}),0)*(1+IFERROR(INDEX(Assumptions!$E:$G,
     MATCH(W{r},Assumptions!$A:$A,0),
     MATCH($L$5,{"Baseline","Upside","Downside"},0)),0)))
```

- Override lookup: `GL Code + Column` (L=Apr, M=May, N=Jun) — no row-position dependency
- If Overrides table has a non-empty value for this GL in this column, it takes priority
- Robust to row insertions, deletions, reordering on either tab

### Conditional Formatting

CF uses `DifferentialStyle` + `Rule(type='expression')` — correct XLSX serialization.

Helper columns X/Y/Z (hidden, cols 24/25/26 on Forecast tab):
```
X{r}: =IF($A{r}="","",COUNTIFS(Overrides!$A$12:$A$400,$A{r},Overrides!$L$12:$L$400,"<>"))
Y{r}: =IF($A{r}="","",COUNTIFS(Overrides!$A$12:$A$400,$A{r},Overrides!$M$12:$M$400,"<>"))
Z{r}: =IF($A{r}="","",COUNTIFS(Overrides!$A$12:$A$400,$A{r},Overrides!$N$12:$N$400,"<>"))
```

CF rules (same-sheet reference — reliable across all Excel versions):
- `L12:L400` → `=$X12>0` → fill gold (#FFE066)
- `M12:M400` → `=$Y12>0` → fill gold
- `N12:N400` → `=$Z12>0` → fill gold

```javascript
return [{

```

## Key Lessons Learned (read before modifying ANY sub-workflow)

1. **n8n field names are case-sensitive.** Always match exact casing in trigger's Workflow Input Schema. Use `$json.PropertyCode`, not `$json.propertyCode`.

2. **ToolWorkflow v2 requires the final node to return a field literally named `output`, as a STRING** (use JSON.stringify). Returning a raw JSON object causes "workflow did not return a response" errors.

3. **"Always Output Data" must be ON on every node in the chain.** Without it, n8n silently halts when zero items are found rather than returning a graceful error.

4. **Large P&L files exceed LLM context limits.** Full GL detail is ~217K tokens — over Claude's 200K limit. Filter to key GL codes and subtotals before returning data.

5. **Use Code node + IF node for found/not-found logic** — not the "Filter" node. Filter's "Kept: 0 items" behaviour doesn't propagate usable data downstream for the false case.

6. **"Respond to Webhook" is wrong for "When Executed by Another Workflow" sub-workflows.** Always use a Code node returning `output` as the final node.

7. **Source file columns 11–13 (Apr/May/Jun) are BUDGET values, not actuals.** Trailing 3-month averages must use cols 8/9/10 (Jan/Feb/Mar).

8. **Override CF requires DifferentialStyle** (openpyxl). Using plain `PatternFill` in `FormulaRule` doesn't serialize CF fills correctly. Use `Rule(type='expression', dxf=DifferentialStyle(...))`.

9. **Override logic must be GL-keyed, not row-position.** Use `COUNTIFS(Overrides!$A$12:$A$400, $A{r}, ...)` — not `IF(Overrides!L{r}<>"", ...)`. Row alignment between tabs is fragile.

10. **Year-suffix file naming.** CY files must NOT include `(2025)` or older year suffixes. PY files must include `(2025)`. Tool 4 v2.5 match logic enforces this.
