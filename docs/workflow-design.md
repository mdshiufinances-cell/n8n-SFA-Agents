## Source File Column Mapping — Dynamic (v3.0)

**⚠️ CRITICAL — Read before modifying any data sourcing logic.**

### Fixed column positions (always true)

| Col index (0-based) | Excel col | Content | Notes |
|---|---|---|---|
| 0 | A | GL Code | Always |
| 1 | B | Account Name | Always |
| 2–13 | C–N | Monthly values — Jul through Jun | 12 months in fiscal year order |
| 14 | O | Total Actual + Budget | Sum of cols 2–13 |
| 15 | P | Original Annual Budget | Full-year budget at time of entry |

### Dynamic split — actuals vs forecast

The column positions above are **fixed**, but whether each monthly column contains an **actual** or a **budget** value depends on when the file was exported from Yardi. All periods up to and including the last closed month contain actuals; remaining months contain budget values.

**Tool 4 v3.0 detects this automatically** using two methods (in priority order):

**Method 1 — Header scan (preferred):** Scans the first 10 rows for a row containing 6+ month-label cells (e.g. "Jul 2025", "Aug 2025"). Compares each period-end date to today's date to classify columns as actual or forecast. Returns `detectionMethod: "header_scan"`.

**Method 2 — Date arithmetic fallback:** If no header row is found, derives the split from today's date and fiscal year start (July 1). Returns `detectionMethod: "date_arithmetic_fallback"`. Add a warning to the forecast output when this fallback is used.

### Tool 4 v3.0 response fields (new)

| Field | Type | Description |
|---|---|---|
| `actualsMonths` | string[] | Month labels with closed actual data — e.g. ["Jul 2025", …, "Mar 2026"] |
| `forecastMonths` | string[] | Open month labels to forecast — e.g. ["Apr 2026", "May 2026", "Jun 2026"] |
| `trailingAvgMonths` | string[] | Last 3 items from `actualsMonths` — used for trailing average |
| `actualsCount` | number | Length of `actualsMonths` |
| `forecastCount` | number | Length of `forecastMonths` |
| `detectionMethod` | string | `"header_scan"` or `"date_arithmetic_fallback"` |

### Example: different export dates for FY2026 (Jul 2025 – Jun 2026)

| File exported | `actualsMonths` | `forecastMonths` | `trailingAvgMonths` |
|---|---|---|---|
| Jan 31, 2026 | Jul 2025 – Jan 2026 (7 months) | Feb – Jun 2026 (5 months) | Nov 2025 / Dec 2025 / Jan 2026 |
| Mar 31, 2026 | Jul 2025 – Mar 2026 (9 months) | Apr – Jun 2026 (3 months) | Jan / Feb / Mar 2026 |
| May 31, 2026 | Jul 2025 – May 2026 (11 months) | Jun 2026 (1 month) | Mar / Apr / May 2026 |
| Jun 30, 2026 | Jul 2025 – Jun 2026 (12 months) | none | Apr / May / Jun 2026 |

### ❌ What NOT to do

- Do NOT assume cols 8/9/10 = actuals — only true for a 9-month actuals file
- Do NOT assume cols 11/12/13 = budget — these may be actuals in later exports
- Do NOT hardcode trailing average month positions — always derive from `trailingAvgMonths`
- Do NOT include `forecastMonths` in YTD calculations

## Tool 4 — Read Yardi Files: Architecture (v3.0)

**Status: ✅ Production — dynamic period detection added**

### Node Structure

```
[When Executed by Another Workflow]
        ↓
[Search files and folders]   (Always Output Data: ON)
        ↓
[Match File]                  (Code node v2.5, Always Output Data: ON)
        ↓
[IF: found === true]
   ↙ true                  ↘ false
[Download file]         [Not Found Message]  (Code node, final node)
   ↓
[Extract from File]   (Always Output Data: ON)
   ↓
[Code in JavaScript]   (v3.0 — dynamic period detection + formats output)
```

### Trigger Input Schema (case-sensitive)

```

FileType      (String)   — actuals | budget | rent_roll | actuals_py | budget_py | rent_roll_py
```

### Search files and folders config

### Match File (Code node v2.5)

```javascript
const triggerData = $('When Executed by Another Workflow').first().json;
const fileType = triggerData.FileType;
const propertyCode = triggerData.PropertyCode;
const items = $input.all();

const patternMap = {
  actuals:      { base: 'actuals',    exclude: 'actualspy'  },
  budget:       { base: 'budget',     exclude: 'budgetpy'   },
  rent_roll:    { base: 'rentroll',   exclude: 'rentrollpy' },
  actuals_py:   { base: 'actualspy',  exclude: null },
  budget_py:    { base: 'budgetpy',   exclude: null },
  rent_roll_py: { base: 'rentrollpy', exclude: null },
};

const config = patternMap[fileType];
if (!config) {
  return [{ json: { found: false, propertyCode, fileType,
    message: `Unknown FileType: ${fileType}` } }];
}

const match = items.find(item => {
  const name = (item.json.name || '').toLowerCase()
    .replace(/_/g, '').replace(/\s/g, '');
  if (!item.json.id) return false;
  if (!name.includes(config.base)) return false;
  if (config.exclude && name.includes(config.exclude)) return false;
  return true;
});

if (match) {
  return [{ json: {
    found: true, id: match.json.id, name: match.json.name, propertyCode, fileType
  }}];
}
return [{ json: { found: false, propertyCode, fileType } }];
```

### Code in JavaScript (TRUE branch, final node — v3.0)

Full source: `code/read-yardi-files.js` in this repo.

Key changes in v3.0 vs v2.5:
- Scans first 10 rows for a month-label header row (e.g. "Jul 2025", "Aug 2025")
- Compares each period-end date to today to classify columns as actual vs forecast
- Falls back to date arithmetic if no header row detected
- Returns named month keys (e.g. `"Jul-25"`) instead of raw column indices
- Adds `actualsMonths`, `forecastMonths`, `trailingAvgMonths`, `actualsCount`, `forecastCount`, `detectionMethod` to every response

```javascript
// Tool 4 — Read Yardi Files — Final Code Node (Output Formatter) v3.0
// KEY CHANGE: Dynamic detection of actuals vs forecast months from file headers

const items = $input.all();

if (!items || items.length === 0 || !items[0].json || Object.keys(items[0].json).length === 0) {
  const propertyCode = $('When Executed by Another Workflow').first().json.PropertyCode;
  const fileType     = $('When Executed by Another Workflow').first().json.FileType;
  return [{ json: { output: JSON.stringify({
    success: false,
    message: `No ${fileType} file found for property ${propertyCode}.`
  }) } }];
}

const fileType     = $('When Executed by Another Workflow').first().json.FileType || 'actuals';
const propertyCode = $('When Executed by Another Workflow').first().json.PropertyCode;
const rows = items.map(item => item.json);

// ── DYNAMIC PERIOD DETECTION ────────────────────────────────────────────────
const MONTH_REGEX = /^(Jan|Feb|Mar|Apr|May|Jun|Jul|Aug|Sep|Oct|Nov|Dec)[\s-]\d{2,4}$/i;
const MONTH_NAMES = ['Jan','Feb','Mar','Apr','May','Jun','Jul','Aug','Sep','Oct','Nov','Dec'];

function parseMonthLabel(raw) {
  const s = String(raw || '').trim().replace(/-/g, ' ');
  const m = s.match(/^(\w+)\s+(\d{2,4})$/);
  if (!m) return null;
  const mIdx = MONTH_NAMES.findIndex(n => n.toLowerCase() === m[1].toLowerCase().slice(0,3));
  if (mIdx === -1) return null;
  const year = m[2].length === 2 ? 2000 + parseInt(m[2]) : parseInt(m[2]);
  return { label: `${MONTH_NAMES[mIdx]} ${year}`, mIdx, year };
}

function monthEndDate(mIdx, year) {
  return new Date(year, mIdx + 1, 0);
}

// Step 1 — find header row by scanning for 6+ month-labelled cells
let headerColMap = null;
for (let i = 0; i < Math.min(rows.length, 10); i++) {
  const vals = Object.entries(rows[i]);
  const found = {};
  let count = 0;
  for (const [key, val] of vals) {
    const parsed = parseMonthLabel(val);
    if (parsed) { found[parseInt(key)] = parsed; count++; }
  }
  if (count >= 6) { headerColMap = found; break; }
}

const today = new Date();
let actualsMonths  = [];
let forecastMonths = [];
let actualsColIdx  = [];
let forecastColIdx = [];

if (headerColMap && Object.keys(headerColMap).length >= 6) {
  const sorted = Object.entries(headerColMap)
    .sort(([a], [b]) => parseInt(a) - parseInt(b));
  for (const [colIdxStr, info] of sorted) {
    const colIdx = parseInt(colIdxStr);
    const end = monthEndDate(info.mIdx, info.year);
    if (end < today) {
      actualsMonths.push(info.label); actualsColIdx.push(colIdx);
    } else {
      forecastMonths.push(info.label); forecastColIdx.push(colIdx);
    }
  }
} else {
  // Fallback: date arithmetic from fiscal year start (July 1)
  const fyStartMonth = 6;
  const fyStartYear  = today.getMonth() >= 6 ? today.getFullYear() : today.getFullYear() - 1;
  for (let m = 0; m < 12; m++) {
    const mIdx  = (fyStartMonth + m) % 12;
    const mYear = fyStartYear + Math.floor((fyStartMonth + m) / 12);
    const label = `${MONTH_NAMES[mIdx]} ${mYear}`;
    const end   = monthEndDate(mIdx, mYear);
    const colIdx = m + 2;
    if (end < today) {
      actualsMonths.push(label); actualsColIdx.push(colIdx);
    } else {
      forecastMonths.push(label); forecastColIdx.push(colIdx);
    }
  }
}

const trailingAvgMonths = actualsMonths.slice(-3);
const detectionMethod   = headerColMap ? 'header_scan' : 'date_arithmetic_fallback';

// ── DATA FILTERING ──────────────────────────────────────────────────────────
const GL_REGEX = /^\d{4,6}$/;
const dataRows = rows.filter(row => {
  const first = String(Object.values(row)[0] || '').trim();
  return GL_REGEX.test(first);
});

const KEY_GL = new Set([
  '40000','41000','41100','41110','41200','41990',
  '42000','42100','42210','42990',
  '43100','43290','43510','44990','45990',
  '60000','60500','69990','75000','75990',
  '80000','87990','89000','89190','89200','89290',
  '89300','89390','90000','90010','99000','99150',
  '99200','99250','99350','99995'
]);

let outputRows;
if (fileType === 'actuals' || fileType === 'budget' ||
    fileType === 'actuals_py' || fileType === 'budget_py') {
  const filtered = dataRows.filter(row => {
    const gl = String(Object.values(row)[0] || '').trim();
    return KEY_GL.has(gl) || gl.endsWith('990') || gl.endsWith('000')
      || gl.endsWith('010') || gl.endsWith('200');
  });
  outputRows = filtered.length > 0 ? filtered : dataRows.slice(0, 80);
} else {
  outputRows = dataRows.slice(0, 300); // rent roll
}

// ── NORMALISE ROW KEYS to named month keys (e.g. "Jul-25") ─────────────────
function shortKey(label) {
  const parts = label.match(/^(\w+)\s+(\d{4})$/);
  if (!parts) return label;
  return `${parts[1].slice(0,3)}-${String(parts[2]).slice(2)}`;
}

const colToKey = {};
[...actualsColIdx, ...forecastColIdx].forEach((ci, i) => {
  const label = i < actualsMonths.length
    ? actualsMonths[i]
    : forecastMonths[i - actualsMonths.length];
  colToKey[ci] = shortKey(label);
});

const namedRows = outputRows.map(row => {
  const out = {};
  for (const [key, val] of Object.entries(row)) {
    const ci = parseInt(key);
    if (ci === 0) { out['gl_code']      = val; continue; }
    if (ci === 1) { out['account_name'] = val; continue; }
    const mk = colToKey[ci];
    if (mk) out[mk] = typeof val === 'number' ? val : (parseFloat(val) || 0);
    if (ci === 15) out['original_budget'] = typeof val === 'number' ? val : (parseFloat(val) || 0);
  }
  return out;
});

return [{ json: { output: JSON.stringify({
  success:          true,
  fileType,
  propertyCode,
  detectionMethod,
  actualsMonths,
  forecastMonths,
  trailingAvgMonths,
  actualsCount:     actualsMonths.length,
  forecastCount:    forecastMonths.length,
  rowCount:         namedRows.length,
  data:             namedRows
}) } }];
```

### Override Formula Pattern (Forecast — forecast month column, row r)

```
=IF(AND($A{r}<>"",
     COUNTIFS(Overrides!$A$12:$A$400,$A{r},Overrides!$[COL]$12:$[COL]$400,"<>")>0),
   SUMIFS(Overrides!$[COL]$12:$[COL]$400,Overrides!$A$12:$A$400,$A{r}),
   IFERROR(AVERAGE([trailing_col1]{r}:[trailing_col3]{r}),0)*(1+IFERROR(INDEX(
     Assumptions!$E:$G,
     MATCH(W{r},Assumptions!$A:$A,0),
     MATCH($[COL]$5,{"Baseline","Upside","Downside"},0)),0)))
```

Where `[COL]` = the Excel column letter for that forecast month, and trailing columns = the 3 most recent actual month columns.

- Override lookup: GL Code keyed — no row-position dependency
- Robust to row insertions, deletions, reordering on either tab

### Conditional Formatting

Helper columns X/Y/Z (hidden): COUNTIFS indicators for each forecast month. CF rules reference same-sheet helper columns for cross-Excel-version reliability.

CF rules: forecast column range → `=$X{r}>0` → fill gold (#FFE066)

## Key Lessons Learned (read before modifying ANY sub-workflow)

1. **n8n field names are case-sensitive.** Always match exact casing in trigger's Workflow Input Schema. Use `$json.PropertyCode`, not `$json.propertyCode`.

2. **ToolWorkflow v2 requires the final node to return a field literally named `output`, as a STRING** (use JSON.stringify). Returning a raw JSON object causes "workflow did not return a response" errors.

3. **"Always Output Data" must be ON on every node in the chain.** Without it, n8n silently halts when zero items are found rather than returning a graceful error.

4. **Large P&L files exceed LLM context limits.** Full GL detail is ~217K tokens — over Claude's 200K limit. Filter to key GL codes and subtotals before returning data.

5. **Use Code node + IF node for found/not-found logic** — not the "Filter" node. Filter's "Kept: 0 items" behaviour doesn't propagate usable data downstream for the false case.

6. **"Respond to Webhook" is wrong for "When Executed by Another Workflow" sub-workflows.** Always use a Code node returning `output` as the final node.

7. **Actuals vs budget split is dynamic — never hardcode column positions.** The Yardi 12-Month Actual Budget report contains actuals for all closed periods and budget for open periods. Which columns are actuals depends on when the file was exported. Tool 4 v3.0 detects this automatically via header scan. Trailing averages must always use `trailingAvgMonths` from the Tool 4 response, not hardcoded positions like "cols 8/9/10".

8. **Override CF requires DifferentialStyle** (openpyxl). Using plain `PatternFill` in `FormulaRule` doesn't serialize CF fills correctly. Use `Rule(type='expression', dxf=DifferentialStyle(...))`.

9. **Override logic must be GL-keyed, not row-position.** Use `COUNTIFS(Overrides!$A$12:$A$400, $A{r}, ...)` — not `IF(Overrides!L{r}<>"", ...)`. Row alignment between tabs is fragile.

10. **File naming convention:** Files use `prop01_actuals.xlsx` format (not original Yardi export names). Match File node v2.5 uses include/exclude pattern matching (e.g. `actuals` included, `actualspy` excluded for CY files). The `.xlss` typo on PY rent roll must be `xlsx`.

11. **Max Iterations must be set to 30** on the AI Agent node (Parameters → Add Option → Max Iterations). The default of 10 is too low for the full 9-step forecasting workflow (5–6 Tool 4 calls + multiple Calculator calls + 1 Tool 5 call = 15–20 iterations minimum).

12. **Google Drive OAuth token expires periodically.** If all Tool 4 FileTypes return not-found simultaneously, reconnect the "Google Drive account" credential in Settings → Credentials before debugging anything else.
